为什么要开这个博客

这个虽然不重要，但还是要啰嗦一下创建本博客初衷，免得过了许久后就不知道当时是怎么个心血来潮建了这个博客，又是怎样渐渐地从开始到放弃的，也好给自己一个说法。

下面开始讲故事。

19年，这个时候部门经理谢哥已经被发放上海平安银行出差一年多了，他临走之前，将这个部门的一切事务压在了我这个技术和业务都不咋地的非科班程序员身上，从此肩负上了部门发展的使命。

自身现状

技术方面：

  • 深度不够。java的底层技术、性能调优、多并发等几乎没涉足。对技术没有深入研究。
  • 广度不够：开发、运维、测试涉及大量的技术使用，如高可用、分布式、分库分表、容器k8s、springboot、springcloud。持续集成、持续部署、DDD领域驱动。。。

业务方面：

  • 目前部门负责的开放平台、开放银行项目，几乎是摸着石头过河。国外有很多案例，但是不符合国情。我们目前只能根据银行现有的需求走，走一步算一步。
  • 自己对金融领域理解不深，来公司的前两年是做电力行业的。对金融领域使用的技术也没有体感。

管理方面：

  • 管人：一个程序员是不太希望老是被别人管的，管理方面一直都认为没必要。6月份上了管理课，才知道管理也有很多东西要学，如何选择人、要求人、辅导人、激励人、评估人、凝聚人。
  • 管项目：也有一大堆的书要学习。项目管理、软件工程、事件风暴、需求管理。

思维方面

  • 系统思维、结构化思维。有时候，你跟一些大佬聊天，就会明显感受到什么叫做降维打击。在多维度思考者面前，单维度的观点简直漏洞百出。

目标

两个方面:

  • 部门负责的项目能够在行业内有较强的竞争力。
  • 自己的能力提升，能够在行业内有较强的竞争力。
    两个目标虽然模糊，但是这两者的方向是一致的，是相互促进的。

如何做

终于要讲到为什么要有这个博客了。
要达到目标，需要不遗余力地学习、实践、总结。
近期发现要学习的东西太多了，于是就有了要将学到的东西汇聚记录到一个地方的诉求。

一开始我写在为知笔记、后来印象笔记，还有一部分记录在公司的个人confluence空间上。终于有一天受不了了，实在是太分散了，不利于学习总结和分享。没有好好管理，到最后，那些之前记录的内容都荒废掉了。

经过一个周末的奋战，终于搭建好了这个博客。Github Pages+Hexo+MWeb的博客写作之路正式开启。

  • Github Pages：用于托管和展示博客内容。相当于博客服务器的角色。
  • Hexo：一款优秀的博客框架。
  • MWeb：用于在本地编写博客。主要是MarkDown格式的文档。

最后，如果你也想搭建一个属于自己的博客，推荐一个博客教程：https://www.simon96.online/。
实在是太心动这个博客的风格了，于是跟这个博客使用了一样的风格。
